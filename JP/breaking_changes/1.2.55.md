# 1.2.55 / 1.2.56

この 2 バージョンで、色・フォント・背景色のカスケードや GridLayout のレンダリングなど、
**レイアウトまわりの見た目に関わる大きめの改善**が入りました。

全体としては表示の一貫性・品質が向上していますが、
**バージョンを上げると既存の画面レイアウトが崩れて見えることがあります。**
大半はデザイナ上でのちょっとした調整（幅・色・フォント・パディング等の指定見直し）で
きれいに収まりますので、アップグレード後に画面を確認して調整してください。

---

## 主な改善内容

細かい修正は多数ありますが、ざっくり以下のカテゴリで変更が入っています。

### 色・フォントまわり
- 背景色 / 文字色 / フォントの**カスケード（親から子への伝わり方）を整理**
- 明示的に指定した値と、親から継承された値を内部で区別し、DOM への二重出力を抑制
- `.card`（`IsBordered: true` の Grid）の背景を基本的に透過に変更
  （下の選択状態や背景画像が透けて見えるようになる）
- ボタンの色・フォント指定が正しく反映されるよう修正
- 検索レイアウトでも背景色が効くように変更
- サイドバー・ヘッダのフォント指定の扱いを調整

### GridLayout まわり
- カラム幅の決定ロジックを整理（`Width` / `MinWidth` / `MaxWidth` / `IsAutoFillWrap` の役割を明確化）
- カラムに最小幅（`MinWidth`）の指定を追加
- 折り返し時のマージン調整
- `FillAvailable`（残領域に伸びる指定）の挙動を調整
- 検索条件の最大値指定など、GridLayout 内で指定できる項目を拡充
- TextField の読み取り専用表示での折り返し調整

### List まわり
- `ListField` / `DetailListField` など List 系コンポーネントへの背景色伝搬を調整
- List 固有のスクロール処理が GridLayout 側に漏れ出していたのを修正

### サイドバー
- 画面幅が狭くなったときの挙動を設定で切り替えできるように変更

### CSS の整理
- 不要になった CSS 変数・CSS クラスを多数削除
- 描画ロジック全般のリファクタリング

---

## アップグレード時に見られやすい変化

実際に画面を開いたときに気付きやすいポイントです。

- カード（枠付き Grid）の背景が、以前は親の色で塗られていたのに透過になって見え方が変わる
- ボタンや見出しに指定していた色・フォントの反映結果が変わる
- 一覧（List）内のセルの背景色が変わる
- GridLayout のカラム幅・折り返し位置が変わる
- 検索レイアウトの背景色が、以前は出ていなかったものが出るようになる
- サイドバー・ヘッダの文字サイズ／フォント指定の反映結果が変わる

---

## 対応方針

基本的には **デザイナ上で該当箇所を見直せば調整できます**。

- 枠付き Grid の背景を以前のように塗りたい場合は、`GridLayoutDesign.BackgroundColor` を明示的に指定
- カラム幅が意図と違って見える場合は、`Width` / `MinWidth` / `MaxWidth` / `IsAutoFillWrap` を指定して調整
- ボタン等の色が想定と違う場合は、対象フィールドの色／フォント指定を確認

大量の画面を一度に直すというよりは、バージョンを上げてから
**違和感のある画面を順次見直していく**方針をおすすめします。
全体としては色・フォント・レイアウトの指定ルールが整理されており、
調整後はこれまでより挙動が読みやすくなっています。
