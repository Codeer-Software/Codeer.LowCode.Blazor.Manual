# 1.3.0

スクリプト言語の型変換ルールを整理・統一しました。
既存スクリプトの実行結果が変わる可能性がある項目の一覧です。

---

## 1. 小数 → 整数型の変換が「切り捨て」になりました

小数（`double` / `float` / `decimal`）を整数型へ変換するとき、これまでは**四捨五入**していましたが、**0 方向への切り捨て**に変わりました（C# のキャストと同じ挙動）。

```csharp
int x = 1.9;       // 変更前: 2（四捨五入） → 変更後: 1（切り捨て）
var y = (int)2.5;  // 変更前: 2 → 変更後: 2
var z = (int)2.9;  // 変更前: 3 → 変更後: 2
```

代入・初期化・キャスト・関数の引数・戻り値のすべてで一貫して切り捨てになります。
四捨五入が必要な場合は `Math.Round(...)` を明示してください。

```csharp
int x = (int)Math.Round(1.9);   // 2
```

---

## 2. 変換できない値の代入・キャストが実行時エラーになりました

変換できない値を **null を取れない型**（`int`、`double` など）へ入れようとすると、**実行時エラー**になります。
これまでは型付き変数への**再代入**だけ、変換に失敗すると黙って `null` になっていました（初期化・キャストは従来からエラー）。挙動を揃えました。

```csharp
int x = 0;
x = "abc";         // 変更前: x は null（無言） → 変更後: 実行時エラー
```

`null` を取れる型（`int?` など）に入れる場合は、これまでどおり `null` になります。

```csharp
int? x = "abc";    // null（変わらず）
```

---

## 3. 文字列 → 数値型のキャストが変換されるようになりました

`string` を数値型へキャストすると、これまでは**変換されずに文字列のまま**返っていました（実質的に無効）。
**変換が行われる**ようになりました。

```csharp
var n = (int)"5";  // 変更前: "5"（文字列のまま） → 変更後: 5
```

数値として読めない文字列は、§2 と同じく非 null 許容型では実行時エラー、null 許容型では `null` になります。

---

## 4. `(int)null` などが既定値になりました

`null` を非 null 許容の値型へキャストすると、**既定値**になります（代入 `int x = null;` が `0` になるのと同じ挙動）。

```csharp
var n = (int)null;   // 変更前: null → 変更後: 0
var m = (int?)null;  // null（変わらず）
```

---

## あわせて改善した点（破壊的ではありません）

- `decimal` と `double` / `float` の `==` / `!=` 比較が実行時エラーにならなくなりました（例: 数値フィールド `== 1.5`）。
- 小数形式の文字列を整数型へ入れると、エラーにならず切り捨てて変換されます（例: `int x = "1.9";` → `1`）。
- スクリプトの型キーワードに `sbyte` を追加しました。
