# HeaderMenuField (ヘッダーメニュー)

## これは何か

**[PageFrame](../designer/page_frame.md) のヘッダーに表示されるメニューを、カスタムモジュールで組み立てるための Field**。
ヘッダーを独自モジュールに置き換えたい場合、そのモジュール内にこの Field を配置すると、標準のヘッダー機能の一部をそのまま使えます。

> 通常は PageFrame の「Header」設定だけでメニューを構築できます。この Field を使うのは、**カスタムモジュールで独自のヘッダー UI を組む**ケースです。PageFrame の `Header.ModuleName` に代替モジュールを指定した時に、この Field を通じて標準メニュー機能を取り込みます。

## いつ使うか

- ヘッダーのレイアウトを独自に組みたい（アイコン配置・独自ボタン・検索ボックスなどを挟みたい）
- 標準では出せないタイミング・条件でユーザー名やログアウトを表示したい

## デザイナでの設定

### プロパティ一覧

#### システム

| C#名 | 日本語表示名 | 説明 |
|---|---|---|
| - | フィールドタイプ | `ヘッダーメニュー` 固定 |

#### 基本設定

| C#名 | 日本語表示名 | 型 | 既定値 | 説明 |
|---|---|---|---|---|
| **Name** | 名前 | string | `""` | フィールド識別子 |
| **Type** | メニュー種別 | enum | `Items` | どの標準パーツを表示するか |
| **IgnoreModification** | 変更判定から除外 | bool | `false` | 変更検知から除外 |

### Type の選択肢

| 値 | 描画される内容 |
|---|---|
| **Home** | PageFrame の `Header.Home` で設定した内容（テキスト / アイコン / 画像） |
| **Items** | PageFrame の `Header.Links` で設定したメニュー項目群 |
| **UserName** | ログイン中のユーザー名 |
| **Logout** | ログアウトボタン |

同じモジュール内に複数配置すれば、それぞれの要素を別々の位置に並べられます。
