# アプリ作成パターン

業務アプリを Codeer.LowCode.Blazor で作るときに繰り返し登場する「型」をまとめたページです。デザインパターンと同じく、**一つひとつは小さい部品**で、組み合わせて実際のアプリを組み立てるためのものです。

新しい画面やデータ構造を作るときに「この場面はどのパターン?」「このパターンは CLB のどの機能で作る?」を引けるリファレンスとして使ってください。

---

## デザイナの標準テンプレート (PatternShowcase) で全パターンを実機確認できる

Visual Studio で新規プロジェクトを作るときの **`PatternShowcase` テンプレート** が、ここで紹介する全パターンの実装サンプル集になっています。

- デザイナで「新規プロジェクト」→ テンプレート選択で **PatternShowcase** を選んで展開
- サイドバーに **データ操作 / リスト / 一覧 / 検索 / レイアウト / 入力UX / 出力 / ダイアログ等 / 別フレーム** のカテゴリで各パターンへのリンクが並んでいる
- 各リンクは「動く画面 + 設定済みモジュール定義」 = そのままコピーして自分のプロジェクトに転用できる

> このマニュアルの各パターンには **PatternShowcase の対応モジュール名**を必ず書いています。マニュアルで概念を掴んでデザイナで実物を触る、という二段構えで読むのがおすすめです。

## パターン一覧

### A. [データ構造のパターン](data_shape.md) ─ モジュールの形 (DB との関係)

| パターン | 用途 |
|---|---|
| 単一 CRUD | 1テーブル＝1モジュール (基本形) |
| ヘッダ詳細 (1:N) | 注文+明細、請求書+行 |
| マスタ参照 (多対1) | 他モジュールから 1件選んで紐付け |
| 多対多 | タグ・属性などの紐付け (中間テーブル) |
| ツリー (自己参照) | カテゴリ階層・組織図 |
| **双方向 ID 持ち合い (1:1)** | 親 ↔ 子 が互いに ID を持つ相互参照 |
| 状態遷移を持つレコード | ステータス管理 (下書き → 申請 → 承認) |
| 履歴・監査 | 変更履歴の自動保存 |
| 論理削除 | 物理削除せず削除フラグ |
| 作成日時・更新日時 | 自動セットされるシステムフィールド |

### B. 画面構成のパターン (執筆予定)

一覧→詳細遷移、検索付き一覧、詳細内の関連リスト、左右分割、ウィザード、ダッシュボード、Detail/Tile 表示など。当面は PatternShowcase の **リスト / 一覧 / レイアウト** カテゴリと [ListPagePatterns.md](../../Claude/ClaudeCodeForDesigner/Docs/ListPagePatterns.md) を参照してください。

### C. 入力支援パターン (執筆予定)

カスケーディング選択、オートサブミット、読取/編集モード切替、コピー新規作成、値の自動計算、入力検証など。PatternShowcase の **入力UX** カテゴリに実装サンプルがあります。

### D. データアクセス・出力パターン (執筆予定)

生 SQL、集計、Excel/PDF 帳票、CSV 一括入出力、WebAPI 連携、メール送信、ファイル添付など。PatternShowcase の **出力 / データ操作/取込書出** カテゴリと [Examples](../Examples/) を参照。

### E. 認証・認可パターン (執筆予定)

ログインユーザー連動、ロール別権限、データ単位権限、マルチテナント。当面は [認証 / 認可の概要](../authorization/authorization.md) を参照。

## パターンの読み方

各パターンページは以下の構造で書いています:

| セクション | 内容 |
|---|---|
| **いつ使う** | 業務シーンの典型例 |
| **DB の形** | テーブルと列・FK の関係 (簡易図) |
| **CLB ではこう作る** | フィールド構成・必要な設定の要点 |
| **PatternShowcase の対応モジュール** | 実機確認用 |
| **落とし穴** | よくある間違い (あれば) |
