# アプリ作成パターン

業務アプリを Codeer.LowCode.Blazor で作るときに繰り返し登場する「型」をまとめたページです。デザインパターンと同じく、**一つひとつは小さい部品**で、組み合わせて実際のアプリを組み立てるためのものです。

新しい画面やデータ構造を作るときに「この場面はどのパターン?」「このパターンは CLB のどの機能で作る?」を引けるリファレンスとして使ってください。

---

## デザイナの「標準パターン集」テンプレートで全パターンを実機確認できる

Visual Studio で新規プロジェクトを作るときに選べる **「標準パターン集」** テンプレート (内部名 `PatternShowcase`) が、ここで紹介する全パターンの実装サンプル集になっています。

- デザイナで「新規プロジェクト」→ テンプレート選択で **標準パターン集** を選んで展開
- サイドバーに **データ操作 / リスト / 一覧 / 検索 / レイアウト / 入力UX / 出力 / ダイアログ等 / 別フレーム** のカテゴリで各パターンへのリンクが並んでいる
- 各リンクは「動く画面 + 設定済みモジュール定義」 = そのままコピーして自分のプロジェクトに転用できる

> このマニュアルの各パターンには **標準パターン集の対応モジュール名**を必ず書いています。マニュアルで概念を掴んでデザイナで実物を触る、という二段構えで読むのがおすすめです。

## パターン一覧

### A. アプリの基本パターン

| パターン | アプリのイメージ |
|---|---|
| [単一 CRUD](single_crud.md) | 商品マスタなど 1 画面で完結する基本形 |
| [ヘッダ詳細 (1:N)](header_detail.md) | 注文+明細、請求書+行 |
| [マスタ参照 (多対1)](lookup.md) | 商品にカテゴリ、注文に顧客を紐付け |
| [多対多](many_to_many.md) | 記事にタグ、ユーザーにロール (中間テーブル) |
| [ツリー (自己参照)](tree.md) | カテゴリ階層・組織図 |
| [**双方向 ID 持ち合い (1:1)**](bidirectional.md) | 会社↔代表者、申請↔承認フロー |
| [状態遷移を持つレコード](state_machine.md) | ステータス管理 (下書き → 申請 → 承認) |
| [履歴・監査](history.md) | 変更履歴の自動保存 |
| [論理削除](soft_delete.md) | 削除フラグで非表示にする |
| [作成日時・更新日時](system_fields.md) | システムフィールドで自動セット |

各パターンは「**アプリの作り** (ユーザーが見る/操作する画面) → **支えるデータ構造** (テーブル) → **モジュールとテーブルの対応** → **CLB ではこう作る**」の流れで読めます。

### B. 画面構成のパターン (執筆予定)

一覧→詳細遷移、検索付き一覧、詳細内の関連リスト、左右分割、ウィザード、ダッシュボード、Detail/Tile 表示など。当面は 標準パターン集の **リスト / 一覧 / レイアウト** カテゴリと [ListPagePatterns.md](../../Claude/ClaudeCodeForDesigner/Docs/ListPagePatterns.md) を参照してください。

### C. 入力支援パターン (執筆予定)

カスケーディング選択、オートサブミット、読取/編集モード切替、コピー新規作成、値の自動計算、入力検証など。標準パターン集の **入力UX** カテゴリに実装サンプルがあります。

### D. データアクセス・出力パターン (執筆予定)

生 SQL、集計、Excel/PDF 帳票、CSV 一括入出力、WebAPI 連携、メール送信、ファイル添付など。標準パターン集の **出力 / データ操作/取込書出** カテゴリと [Examples](../Examples/) を参照。

### E. 認証・認可パターン (執筆予定)

ログインユーザー連動、ロール別権限、データ単位権限、マルチテナント。当面は [認証 / 認可の概要](../authorization/authorization.md) を参照。

## パターンの読み方

各パターンは以下の構造で書いています:

| セクション | 内容 |
|---|---|
| **いつ使う** | 業務シーンの典型例 |
| **アプリの作り** | ユーザーが見る画面と操作の流れ |
| **支えるデータ構造** | テーブルと列・FK の関係 (簡易図) |
| **モジュールとテーブルの対応** | どのモジュールがどのテーブル/フィールドを持つか |
| **CLB ではこう作る** | フィールド構成・必要な設定の要点 |
| **標準パターン集の対応** | 実機確認用のサイドバーリンク + モジュール名 |
| **落とし穴** | よくある間違い (あれば) |
